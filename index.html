What is a VLAN?

A Virtual Local Area Network (VLAN) is a logical subdivision of a physical network that creates separate broadcast domains.

Key Benefits

  • Broadcast Domain Isolation: Reduces network congestion
  • Enhanced Security: Isolates sensitive traffic
  • Improved Performance: Smaller broadcast domains
  • Flexible Design: Logical grouping regardless of location
  • Cost Efficiency: Multiple logical networks on one switch

VLAN Types

Data VLANs: User-generated traffic
Voice VLANs: VoIP traffic with QoS
Management VLANs: Administrative access
Native VLANs: Untagged trunk traffic

Without VLANs vs With VLANs

Traditional Network

All ports = one broadcast domain
Broadcasts reach every device
No logical separation

VLAN Network

Each VLAN = separate broadcast domain
Broadcasts contained within VLAN
Logical separation enforced

Example: 24-Port Switch

Ports 1-8:   VLAN 10 (Sales)
Ports 9-16:  VLAN 20 (Engineering)  
Ports 17-24: VLAN 30 (Management)

üîó Lesson 4: Access vs Trunk Ports

Access Ports

  • Connect end devices
  • Carry ONE VLAN only
  • Send/receive untagged frames
  • Add tags when forwarding to trunk
Use: PCs, printers, servers, APs

Trunk Ports

  • Connect switches/routers
  • Carry MULTIPLE VLANs
  • Use VLAN tags to identify traffic
  • Native VLAN traffic untagged
Use: Inter-switch links, router connections

Frame Handling Example

Access Port: PC ‚Üí Switch (untagged) ‚Üí Add tag ‚Üí Trunk
Trunk Port:  Switch A ‚Üí Tagged frames ‚Üí Switch B ‚Üí Remove tag

Native VLAN Concept

The Native VLAN handles untagged traffic on trunk ports.

  • Default: VLAN 1 on Cisco switches
  • Native VLAN frames sent untagged across trunk
  • Both trunk ends must have same native VLAN

‚ö†Ô∏è Security Risks

  • VLAN 1 is default native VLAN
  • Management traffic often uses VLAN 1
  • Potential for VLAN hopping attacks

‚úÖ Best Practices

  1. Change native VLAN from default VLAN 1
  2. Use unused VLAN as native (e.g., VLAN 99)
  3. Don't assign access ports to native VLAN
  4. Configure native VLAN consistently on both ends

‚öôÔ∏è Lesson 6: Cisco VLAN Configuration

Basic VLAN Creation

Switch> enable
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name SALES
Switch(config-vlan)# exit

Access Port Configuration

Switch(config)# interface range FastEthernet0/1-8
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 10
Switch(config-if-range)# spanning-tree portfast
Switch(config-if-range)# no shutdown

Trunk Port Configuration

Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport trunk native vlan 99
Switch(config-if)# switchport trunk allowed vlan 10,20,99

Verification Commands

show vlan brief
show interfaces trunk
show interfaces Fa0/1 switchport
show mac address-table vlan 10

üõ£Ô∏è Lesson 7: Inter-VLAN Routing

Why Inter-VLAN Routing?

VLANs create separate broadcast domains. Devices in different VLANs cannot communicate without a router.

Method 1: Router on a Stick

Router(config)# interface GigabitEthernet0/1
Router(config-if)# no ip address
Router(config-if)# no shutdown

Router(config)# interface GigabitEthernet0/1.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0

Router(config)# interface GigabitEthernet0/1.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0

Method 2: Switch Virtual Interfaces (SVI)

Switch(config)# ip routing
Switch(config)# interface vlan 10
Switch(config-if)# ip address 192.168.10.1 255.255.255.0
Switch(config-if)# no shutdown

üìã Lesson 8: VLAN Design Best Practices

VLAN Planning

Documentation

Maintain VLAN ID assignments and naming conventions

VLAN Ranges

1-99: Infrastructure
100-199: User VLANs
200-299: Voice VLANs
300-399: Guest/DMZ

Naming

Use descriptive names: SALES_VLAN, ENG_VLAN

Security Best Practices

  1. Change default native VLAN
  2. Prune unnecessary VLANs from trunks
  3. Disable unused ports
  4. Implement VLAN access control

Performance Considerations

  • Right-size VLANs (avoid too large broadcast domains)
  • Strategic trunk placement
  • Load balancing across multiple trunks

üîß Lesson 9: Troubleshooting VLANs

Common Issues

1. VLAN Mismatch

Symptoms: No connectivity between switches

Cause: Different native VLANs on trunk ends

! Check and fix
show interfaces trunk
switchport trunk native vlan 99

2. Missing VLAN on Trunk

Symptoms: VLAN works locally but not across trunk

! Add missing VLAN
switchport trunk allowed vlan add 30

3. Wrong Access VLAN

Symptoms: Device can't communicate with VLAN members

! Check and correct
show interfaces Fa0/5 switchport
switchport access vlan 10

Troubleshooting Toolkit

show vlan brief
show interfaces trunk
show mac address-table
show spanning-tree vlan [number]
debug sw-vlan packets
